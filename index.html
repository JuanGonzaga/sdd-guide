Claude Code + SDD + Speckit

De vibe coding para engenharia orientada por especificação.

Um fluxo operacional para transformar cards do Jira em código implementado, validado e revisável, sem suposições, sem caixa-preta e sem modernizar legado onde não deve.

        O problema
        
IA sem método só acelera a bagunça.


        

          

            ?
            
Regra inventada

            
O agente preenche lacunas sozinho quando a demanda não está especificada.

          


          

            ⚠
            
Legado quebrado

            
Sem contexto, ele pode modernizar PHP 7.2/Zend onde deveria apenas seguir padrão existente.

          


          

            ↻
            
Retrabalho no PR

            
O código nasce rápido, mas volta com bug, inconsistência ou regra mal interpretada.

          

        

      


      

        A ideia central
        
Três peças, um sistema.


        

          

            01
            
Claude Code

            
É o agente. Lê arquivos, executa comandos, edita código e navega no repositório.

          


          

            02
            
SDD

            
É o método. Antes do código, existe especificação, plano, tasks e validação.

          


          

            03
            
Speckit

            
É a ferramenta. Transforma o método em comandos executáveis dentro do Claude Code.

          

        

      


      

        Fluxo operacional
        
Do card Jira ao código validado.


        

          
          
          
          
          
          
          
          
        


        

          

          

        

      


      

        Speckit
        
As skills que tornam o SDD executável.


        

          
/speckit-specify
Cria a spec focada no WHAT, não no HOW.

          
/speckit-clarify
Encontra ambiguidades antes do plano técnico.

          
/speckit-plan
Gera plano, arquitetura, contratos e decisões técnicas.

          
/speckit-tasks
Quebra o trabalho em tasks atômicas e ordenadas.

          
/speckit-implement
Executa as tasks e marca progresso.

          
/speckit-analyze
Audita consistência entre spec, plan e tasks.

          
/speckit-checklist
Gera checklist de validação da feature.

          
/speckit-constitution
Define princípios permanentes do projeto.

          
/speckit-taskstoissues
Converte tasks em GitHub Issues.

        Skills Superlógica
        
Contexto do legado vira regra operacional.


        

          
legacy-project-context
Obriga o agente a buscar exemplos no código antes de propor solução.

          
controller-rules
Garante controllers finos, delegação para service e resposta padronizada.

          
service-repository
Separa regra de negócio de acesso a dados.

          
db-conventions
Protege nomenclatura, queries, bind e multi-tenant.

          
frontend-flow
Respeita Forms, Grids, modais e JS legado.

          
encoding-safe
Evita corromper arquivos ISO-8859-1.

        Uso no dia a dia
        
Prompt base para iniciar uma demanda.


        

          

Objetivo:
Implementar a demanda seguindo SDD, Speckit e as skills do projeto.

Fluxo obrigatório:
1. Ler o card/contexto.
2. Gerar ou revisar o PRD em ~/SDD/.
3. Rodar specify.
4. Rodar clarify se houver ambiguidade.
5. Rodar plan.
6. Rodar tasks.
7. Implementar apenas depois das tasks.
8. Validar com checklist/analyze.
9. Revisar diff antes do PR.

Regras:
- Não inventar regra de negócio.
- Não modernizar legado sem necessidade.
- Buscar exemplos no código antes de implementar.
- Respeitar PHP 7.2, Zend Framework 1 e padrões do projeto.
- Explicar no final o que mudou, como testar e riscos.
